# 60 - Cargo Workspace Setup

**Output**: `{project_root}/Cargo.toml`, `{project_root}/crates/*/Cargo.toml`

## Purpose

Generate a complete Cargo workspace configuration from the migration plan. This scaffold gives the team a compilable Rust project structure that mirrors the source codebase's module organization, ready for dev-workflow to fill in implementations.

## Prerequisites

Before running this guide, these files must exist:
- `.migration-plan/mappings/module-mapping.md` -- defines Rust crate/module structure
- `.migration-plan/mappings/dependency-mapping.md` -- defines crate dependencies
- `.migration-plan/mappings/error-hierarchy.md` -- defines shared error types

## Method

### Step 1: Read migration mappings

Read these files:
- `module-mapping.md` -- extract: crate names, module hierarchy, inter-crate dependencies
- `dependency-mapping.md` -- extract: external crate names and versions for each module
- `error-hierarchy.md` -- extract: whether a shared `error` crate is needed

Build a dependency graph:
```
models (no internal deps, external: serde, chrono, uuid)
error (depends on: models; external: thiserror, axum for IntoResponse)
services (depends on: models, error; external: sqlx, reqwest)
api (depends on: models, error, services; external: axum, tower-http)
config (no internal deps; external: dotenvy, config-rs)
```

### Step 2: Generate workspace Cargo.toml

Create the root `Cargo.toml` with workspace definition:

```toml
[workspace]
members = [
    "crates/models",
    "crates/error",
    "crates/config",
    "crates/services",
    "crates/api",
]
resolver = "2"

[workspace.package]
version = "0.1.0"
edition = "2024"
rust-version = "1.85"
license = "MIT"

[workspace.dependencies]
# Shared dependencies -- declare version once, reference in members
serde = { version = "1", features = ["derive"] }
serde_json = "1"
tokio = { version = "1", features = ["full"] }
tracing = "0.1"
tracing-subscriber = { version = "0.3", features = ["env-filter", "json"] }
thiserror = "2"
anyhow = "1"
chrono = { version = "0.4", features = ["serde"] }
uuid = { version = "1", features = ["v4", "serde"] }

# Web
axum = "0.8"
tower = "0.5"
tower-http = { version = "0.6", features = ["cors", "trace", "compression-gzip"] }

# Database
sqlx = { version = "0.8", features = ["runtime-tokio", "postgres", "chrono", "uuid"] }

# Internal crates
models = { path = "crates/models" }
error = { path = "crates/error" }
config = { path = "crates/config" }
services = { path = "crates/services" }
api = { path = "crates/api" }
```

### Step 3: Generate member crate Cargo.toml files

For EACH crate identified in module-mapping.md, create its `Cargo.toml`:

```toml
# crates/models/Cargo.toml
[package]
name = "models"
version.workspace = true
edition.workspace = true

[dependencies]
serde.workspace = true
chrono.workspace = true
uuid.workspace = true

[dev-dependencies]
serde_json.workspace = true
```

Rules:
- Use `workspace = true` for shared dependencies
- Only include dependencies the module actually needs
- Add `[dev-dependencies]` for testing-specific crates
- Set `default-run` on the binary crate if applicable

## Template

The workspace structure follows this pattern:

```
Workspace Type: {monorepo with internal crates | single crate}
Total Crates: {N}
Binary Crate: {crate name, if web server or CLI}
Library Crates: {comma-separated list}

Crate Dependency Order (build order):
1. {crate} -- no internal deps
2. {crate} -- depends on: {list}
3. {crate} -- depends on: {list}
...
```

## Verification

After scaffold generation, verify:
```bash
# Must compile with no errors
cargo build --workspace

# Must have no warnings
cargo clippy --workspace -- -D warnings

# Must format cleanly
cargo fmt --check

# Structure check
cargo metadata --format-version 1 | jq '.workspace_members'
```

## Quality Criteria

- [ ] All crate names follow Rust naming conventions (snake_case)
- [ ] Workspace dependencies declared once, referenced with `workspace = true`
- [ ] Every crate compiles independently (`cargo build -p {crate}`)
- [ ] No circular dependencies between crates
- [ ] Binary crate has `main.rs`, library crates have `lib.rs`
- [ ] Internal crate dependencies match module-mapping.md
- [ ] External crate versions match dependency-mapping.md
- [ ] `cargo build --workspace` succeeds
